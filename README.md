Лабораторна робота 8, Комар Марія, ТК-31
Варіант 4:
"Побудувати Lex-специфікацію, що обробляє одинокий текстовий файл. За специфікацією має отримуватися лексичний аналізатор, що породжує потік
лексем для мови виразів, що складаються з ідентифікаторів (латиниця), логічних
значень 0 та 1, дужок ( та ), операторів and, or, not та білих символів. Білі символи є
роздільниками та лексеми не утворюють.
Елементом вихідного потоку має бути тип лексеми (число) та сама лексема.
Нумерацію типів лексем задати самостійно."

Нумерація лексем:
1 -ідентифікатор 
2 - логічні значення 0 або 1 
3 - ( 
4 - ) 
5 - and 
6 - or 
7 - not 

Файли:
lexer.l - основний файл, де описані всі правила розпізнавання лексем. Flex генерує з нього файл lex.yy.c
lex.yy.c - згенерований C-код лексичного аналізатора. Компілюється у виконуваний файл.
input.txt - вхідний файл, для прикладу, з виразами, які потрібно проаналізувати.

Для збірки та запуску проєкту потрібно виконати наступні команди:
flex lexer.l
gcc lex.yy.c -o lexer
lexer input.txt
